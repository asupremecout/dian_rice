## 项目目标

实现一个名为 `mini-agent` 的终端程序。

程序启动后进入 ncurses 界面，界面从上到下分为三部分：

```Plain Text

│ 对话区                                         │
│ 显示用户输入、系统回复、工具调用结果                │

├────────────────────────────────────────┤
│ > 用户输入区                                    │
│                                               │
│                                               │
```

输入区固定占用窗口底部 3 行。状态栏位于输入区上方 1 行。对话区占用状态栏以上的区域。

# Level 2 \- 手动命令

在这一阶段，你需要实现几条本地命令：

```Plain Text
/read

/exec
```

这些命令暂时不依赖真正的大模型。用户在输入区输入命令后，程序解析命令，调用对应本地工具，并将结果显示到对话区。

---

## 2\.1 读取指令：/read

命令格式：

```Plain Text
/read <file> [<lines>]
```

示例：

```Plain Text
/read README.md
/read README.md 20
/read README.md 40-60
```

### 行为

1. 如果指定了行数，则读取文件前 N 行。

2. 如果没有指定行数，则尝试读取整个文件。

3. 如果没有指定行数且文件超过 100 行，拒绝读取。

4. 如果指定行数超过 500 行，拒绝读取。

5. 如果文件不存在，返回错误。

6. 如果权限不足，返回错误。

7. 如果文件不是文本文件，拒绝读取。（进阶）

### 文本文件判定（进阶）

`file` 可以用于识别文件是否为文本。如：

```JSON
$ file -b --mime-type texput.log 
text/plain
```

那么可以通过`execlp("file", "file", "--mime-type", "-b", path, NULL);`判断是否以 `text/`开头确定文件类型。

### 成功显示示例

```Plain Text
Tool_Use [read_file] file=README.md lines=1-20

File README.md Read success, Line 1-20

<文件内容>
```

### 失败显示示例

```Plain Text
Tool_Use [read_file] file=README.md

File README.md Read failed, not found
```

失败的原因还有可能是权限不足，非文本文件

## 2\.3 执行指令：/exec

命令格式：

```Plain Text

```

示例：


### 行为

1. 执行给定的非交互命令。

2. 命令没有交互式输入。

3. 捕获命令输出，并显示到对话区。

4. 命令超时时间为 10 秒。

5. 返回命令退出状态。

6. 基础版本可以将 stdout 和 stderr 合并显示。

7. 输出过长时可以截断，建议最多显示 16KB。

### 成功显示示例


### 失败显示示例

```Plain Text

Command exited with code 2

<命令输出>
```

### 超时显示示例

```Plain Text

Command timeout after 10s
```

进阶要求：命令超时后，应能终止子进程组，包括可能产生的孙进程。

# Level 3 \- 历史记录

在这一阶段，你需要把用户输入、系统回复和工具调用结果记录到历史文件中。

建议使用 JSONL 格式。JSONL 格式是指每行都是一个完整的 JSON 块，因此解析需要按行读取。

历史文件路径固定为：`log/history.jsonl`

记录不是读取哦。

---

## 3\.1 用户输入与固定回复记录

当用户输入普通文本时，应记录用户消息和 assistant 的固定回复。

示例：

```JSON
{"role":"user","message":"hello"}
{"role":"assistant","message":"## `429` Too many Requests\n\n**服务器繁忙，请稍后再试。**"}
```

要求：

1. 每条记录是一行 JSON。

2. 字符串需要正确转义。

3. 程序退出后，历史文件仍然存在。

4. 程序重新启动时，可以读取历史文件，并显示最近若干条消息。

不要重新执行工具。

---

## 3\.2 读取指令记录

当执行 `/read` 时，应记录用户输入、工具调用和工具结果。

成功示例：

```JSON
{"role":"user","message":"/read README.md 20"}
{"role":"assistant","message":"Tool_Use read_file","tool_call":{"tool":"read_file","params":{"file":"README.md","lines":20}}}
{"role":"tool","message":"File README.md Read success, Line 1-20","tool":"read_file","result":{"ok":true,"data":{"file":"README.md","start_line":1,"end_line":20,"text":"..."}}}
```

失败示例：

```JSON
{"role":"user","message":"/read missing.txt"}
{"role":"assistant","message":"Tool_Use read_file","tool_call":{"tool":"read_file","params":{"file":"missing.txt"}}}
{"role":"tool","message":"File missing.txt Read failed, not found","tool":"read_file","result":{"ok":false,"error":{"code":"not_found","message":"file not found"}}}
```

常见错误类型：

```Plain Text
not_found
permission_denied
not_text
too_many_lines
invalid_argument
```

---

## 3\.3 写入指令记录

当执行 `/write` 时，应记录用户输入、工具调用和工具结果。

成功示例：

```JSON
{"role":"user","message":"/write note.txt hello"}
{"role":"assistant","message":"Tool_Use write_file","tool_call":{"tool":"write_file","params":{"file":"note.txt","content":"hello"}}}
{"role":"tool","message":"File note.txt Write success","tool":"write_file","result":{"ok":true,"data":{"file":"note.txt","bytes":6}}}
```

失败示例：

```JSON
{"role":"user","message":"/write /root/note.txt hello"}
{"role":"assistant","message":"Tool_Use write_file","tool_call":{"tool":"write_file","params":{"file":"/root/note.txt","content":"hello"}}}
{"role":"tool","message":"File /root/note.txt Write failed, permission denied","tool":"write_file","result":{"ok":false,"error":{"code":"permission_denied","message":"permission denied"}}}
```

要求：

1. `params.content` 需要正确转义。

2. 写入成功时记录写入字节数。

3. 写入失败时记录错误原因。

## 3\.4 执行指令记录

当执行 `/exec` 或 `/exec_input` 时，应记录用户输入、工具调用和工具结果。

在记录上，二者都是 `exec_cmd`，没有区别。

成功示例：

```JSON
{"role":"user","message":"/exec ls"}
{"role":"assistant","message":"Tool_Use exec_cmd","tool_call":{"tool":"exec_cmd","params":{"cmd":"ls","timeout":10}}}
{"role":"tool","message":"Command exited with code 0","tool":"exec_cmd","result":{"ok":true,"data":{"cmd":"ls","exit_code":0,"output":"..."}}}
```

失败示例：

```JSON
{"role":"user","message":"/exec make"}
{"role":"assistant","message":"Tool_Use exec_cmd","tool_call":{"tool":"exec_cmd","params":{"cmd":"make","timeout":10}}}
{"role":"tool","message":"Command exited with code 2","tool":"exec_cmd","result":{"ok":false,"data":{"cmd":"make","exit_code":2,"output":"..."}}}
```

超时示例：

```JSON
{"role":"user","message":"/exec ./loop"}
{"role":"assistant","message":"Tool_Use exec_cmd","tool_call":{"tool":"exec_cmd","params":{"cmd":"./loop","timeout":10}}}
{"role":"tool","message":"Command timeout after 10s","tool":"exec_cmd","result":{"ok":false,"error":{"code":"timeout","message":"command timeout"}}}
```

# 调试建议

如果 ncurses 程序崩溃后终端显示异常，可以尝试：

```Bash
reset
```

如果程序无法退出，可以尝试在另一个终端中结束进程。

调试 ncurses 程序时，可以将日志写入文件，例如：

```Plain Text
log/debug.log
```

不要直接在 ncurses 程序运行时使用 `printf()` 调试界面逻辑，否则可能破坏屏幕显示。

# 附录 B：命令格式总结

```Plain Text
/exit
退出程序。

/read <file> [<lines>]
读取文本文件。未指定行数且文件超过 100 行时拒绝读取。

/write <file> <content>
如果文件不存在则新建；如果文件存在则追加一行内容。

/exec <cmd>
执行非交互命令，捕获输出，超时 10 秒。

/exec_input <cmd>
进阶功能。执行需要输入的命令，可以临时切出 ncurses。
```
